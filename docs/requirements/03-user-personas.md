# 用户画像与场景

> 📖 **相关文档**：[产品需求文档 (PRD)](../PRD.md) | [概述](./01-overview.md) | [产品功能模块](./02-features.md)

## 目标用户画像

CrossMind 主要面向两类用户：Solo Entrepreneurs 和小型创业团队。

---

## 主要用户群（Primary Persona）

### 角色：Solo Entrepreneur - Sarah

#### 基本信息

- **年龄**：28-35 岁
- **职业背景**：产品经理、设计师、营销人员
- **技术水平**：懂基本概念，但不会编程
- **创业阶段**：0-1 阶段，想法验证期
- **工作地点**：远程/咖啡厅/共享办公空间
- **工具栈**：Notion、Figma、Google Suite、社交媒体

#### 痛点

1. **想法管理混乱**
   - 有很多想法，但散落在笔记本、便签、脑海中
   - 不知道如何系统化整理和评估想法优先级
   - 缺乏结构化的思维框架

2. **调研能力受限**
   - 需要做市场调研，但没时间和精力手动搜集
   - 不知道如何科学地验证想法可行性
   - 难以找到目标用户的真实反馈

3. **技术门槛高**
   - 想做 MVP，但雇佣开发团队成本太高（$5000-$20000）
   - 不懂代码，无法自己实现原型
   - 外包质量难以保证，沟通成本高

4. **工具复杂度不匹配**
   - 项目管理工具太复杂，一个人用 Jira 感觉"杀鸡用牛刀"
   - 需要在多个工具间切换（笔记、任务、文档、设计）
   - 缺乏整体视角，难以追踪项目全局进度

#### 目标

- **短期目标**（3-6 个月）：
  - 验证 3 个想法，找到最有潜力的方向
  - 完成 MVP 并获得 100 个早期用户反馈
  - 保持低成本（< $1000/月）

- **中期目标**（6-12 个月）：
  - 找到 Product-Market Fit (PMF)
  - 达到 $1000 MRR（月经常性收入）
  - 考虑融资或招人

- **工作方式偏好**：
  - 希望把 80% 时间花在"思考"而非"执行"上
  - 喜欢异步工作，不喜欢频繁开会
  - 重视自动化和效率工具

#### 使用场景

**周一上午：梳理本周计划**
```
- 打开 CrossMind，查看上周进展
- 在 Canvas 上更新本周要验证的新想法
- 与 AI 对话，完善想法的核心假设
- 创建本周任务列表
```

**周二下午：深入调研**
```
- 在 Agent 雇佣中心选择"Reddit 调研"服务
- 输入关键词和目标社区
- 支付 $29，等待 Agent 工作
- 1-2 小时后收到调研报告
```

**周三全天：完善商业模式**
```
- 基于调研结果，在 Canvas 上应用"商业模式画布"模板
- AI 自动填充部分模块（基于调研数据）
- 手动调整和补充细节
- 导出 PDF 分享给朋友征求意见
```

**周四上午：生成 Landing Page**
```
- 雇佣"Waitlist Landing Page Generator" Agent
- 填写产品名称、核心卖点、品牌色调
- 支付 $39
- 2 小时后收到预览链接，提供 2 次反馈
- 批准部署到生产环境
```

**周五下午：追踪进展**
```
- 查看任务中心，本周完成 5/7 任务
- 与 AI 对话："帮我总结本周进展"
- AI 生成周报（保存到知识库）
- 规划下周优先级
```

#### 成功指标

- **使用频率**：每周至少登录 3 次
- **项目进展**：3 个月内完成 1 个完整的"想法 → MVP"流程
- **成本节约**：相比传统方式节省 70% 以上成本
- **满意度**：NPS > 50

## 次要用户群（Secondary Persona）

### 角色：小型创业团队 - TechBros

#### 基本信息

- **团队规模**：2-5 人
- **角色分工**：
  - 技术创始人（CTO）：负责技术架构和开发
  - 产品/运营（CEO）：负责产品方向和增长
  - （可选）设计师：负责 UI/UX
- **技术水平**：至少有一人懂开发（全栈或前端+后端）
- **创业阶段**：已有 MVP，进入快速迭代期
- **融资状态**：Pre-seed 或天使轮
- **工具栈**：GitHub、Linear、Figma、Notion、Slack

#### 痛点

1. **时间分配困难**
   - 团队小，每个人身兼多职，时间不够用
   - 技术创始人既要写代码，又要做产品决策
   - 产品创始人既要管运营，又要搞增长

2. **文档更新不及时**
   - 需要频繁调整产品方向，但文档更新跟不上
   - PRD、技术文档、API 文档散落各处
   - 新成员 onboard 时找不到最新信息

3. **执行力瓶颈**
   - 想尝试很多增长策略，但执行力有限
   - 需要外包部分工作（设计、内容、营销），但管理成本高
   - 希望用 AI 自动化非核心工作

4. **协作效率低**
   - 协作工具太多（GitHub、Linear、Slack、Notion、Figma）
   - 信息散落各处，难以形成统一视图
   - 会议时间占比过高（30-40%）

#### 目标

- **短期目标**（3 个月）：
  - 提升迭代速度，从 2 周一个版本到 1 周一个版本
  - 用 AI 自动化 50% 的非核心工作
  - 保持团队 focus，减少工具切换时间

- **中期目标**（6-12 个月）：
  - 达到 $10K MRR
  - 团队扩展到 10 人
  - 建立标准化的工作流程

- **工作方式偏好**：
  - 偏好异步协作，减少会议
  - 重视文档和知识沉淀
  - 希望工具能自动化跟踪进度

#### 使用场景

**周一上午：团队周会**
```
创始人 A（CEO）：
- 打开 CrossMind，在 Canvas 上更新本周 OKR
- 创建 3 个 Objective，每个包含 2-3 个 Key Result
- 系统自动将 KR 转化为任务中心的 Task
- AI 基于历史数据，预测任务工作量

创始人 B（CTO）：
- 实时看到 Canvas 更新（WebSocket 同步）
- 在任务中心查看分配给自己的任务
- 发现"技术架构优化"任务缺少验收标准
- 在任务评论中 @AI："帮我写一下这个任务的验收标准"
```

**周二下午：异步协作**
```
创始人 B（CTO）：
- 完成任务"添加用户认证"
- 在任务评论中 @创始人 A："已完成，帮忙 Review 一下"

创始人 A（CEO）：
- 收到通知，查看任务
- 发现需要小调整，在评论中回复建议
- 创始人 B 收到通知，修改代码
- 重新提交 Review，创始人 A 批准
- 任务状态自动更新为 Done
```

**周三全天：雇佣 Agent 做增长实验**
```
创始人 A（CEO）：
- 想测试 3 个增长渠道（Reddit、Product Hunt、Twitter）
- 在 Agent 雇佣中心选择"Social Media Campaign"服务
- 填写目标、预算、时间线
- Agent 自动生成 Campaign 策划文档
- 拆解成 15 个可执行任务，分配到任务中心
- 部分任务分配给创始人 A，部分分配给 Agent
```

**周五下午：复盘会议**
```
创始人 A & B：
- 打开 Canvas"本周 OKR"节点
- AI 自动显示进度统计：
  - 完成任务：8/10
  - 阻塞任务：1（等待第三方 API）
  - 超预期任务：1（用户注册转化率提升 15%）
- 讨论下周计划
- AI 根据本周数据，推荐下周任务优先级
```

#### 成功指标

- **使用频率**：每天至少登录 1 次
- **协作效率**：会议时间减少 30%
- **迭代速度**：版本发布周期缩短 50%
- **团队满意度**：工具 NPS > 60

## 核心使用场景详解

### 场景 1：想法孵化（Ideation → Research）

#### 用户故事

> "作为一个有想法的 solo entrepreneur，我希望能够快速验证我的想法是否值得投入时间，所以我需要一个工具帮我整理思路并做初步市场调研。"

#### 详细流程

**1. 初始输入（5 分钟）**

```
用户：在 CrossMind 新建项目"AI 驱动的健身教练 App"
    ↓
用户：在对话框中描述想法：
      "我想做一个 App，用 AI 根据用户的身体数据和目标生成个性化健身计划"
```

**2. AI 辅助梳理（10 分钟）**

```
AI：自动提问引导用户思考
    "我看到你创建了一个新想法：AI 驱动的健身教练 App。
     让我帮你完善这个想法吧！请告诉我：
     1. 你的目标用户是谁？（健身新手、中级爱好者、专业运动员？）
     2. 你想解决什么核心痛点？
     3. 类似产品有哪些？你的差异化在哪？"
    ↓
AI：将对话内容自动整理成 Canvas 节点
    - 节点 1：目标用户 - "25-35 岁健身新手，想减脂但不知道从哪开始"
    - 节点 2：核心痛点 - "通用计划效果差，私教太贵"
    - 节点 3：解决方案 - "AI 生成个性化计划 + 实时反馈调整"
```

**3. 触发调研 Agent（1 小时自动执行）**

```
用户：在 Agent 雇佣中心选择"Reddit 社区调研"服务
    ↓
用户：输入关键词"fitness app, workout plan, beginner"
    ↓
Agent：自动搜索相关讨论，提取高频痛点和需求
    ↓
Agent：生成调研报告，保存到知识库，并在 Canvas 上创建"调研发现"节点
```

**4. 完善商业模式（30 分钟）**

```
用户：基于调研结果，在 Canvas 上使用"商业模式画布"模板
    ↓
AI：根据调研数据，自动填充"用户痛点""竞品分析""价值主张"等模块
    ↓
用户：手动调整和补充细节
    ↓
结果：一个完整的商业模式 Canvas
```

#### 预期产出

- Canvas 上有 15-20 个结构化节点
- 一份 5-10 页的调研报告（PDF + Markdown）
- 高频痛点清单（按提及次数排序）
- 任务中心中自动生成 3-5 个后续任务（如"完善 UX 设计""技术选型"等）

### 场景 2：原型搭建（Prototype）

#### 用户故事

> "作为一个非技术创始人,我希望能快速搭建一个 MVP 来收集用户反馈，所以我需要 AI 帮我生成 Landing Page 和基础功能原型。"

#### 详细流程

**1. 确定 MVP 范围（15 分钟）**

```
用户：在 Canvas 上创建"MVP 功能清单"节点
    ↓
用户：与 AI 对话："我想先做一个 Waitlist 页面验证需求，需要包含什么？"
    ↓
AI：给出建议清单
    - Hero Section（价值主张 + CTA）
    - Feature Highlights（3 个核心卖点）
    - Social Proof（如果有的话）
    - Email Capture Form
    - FAQ
    ↓
用户：确认清单，AI 自动创建任务"设计 Landing Page"
```

**2. 雇佣 Agent 生成原型（2 小时自动执行）**

```
用户：在 Agent 雇佣中心选择"Waitlist Landing Page Generator"
    ↓
用户：填写输入
    - 产品名称：FitAI
    - 核心卖点：个性化 AI 健身计划，价格比私教便宜 80%
    - 目标用户：健身新手
    - 品牌色调：活力橙 + 科技蓝
    ↓
Agent：自动执行
    - 生成 Next.js + TailwindCSS 代码
    - 集成 Resend/Mailchimp 收集邮箱
    - 配置 Vercel 部署
    - 生成预览链接
    ↓
Agent：在任务中心更新状态"已完成代码生成"
```

**3. 迭代优化（最多 5 次）**

```
用户：查看预览链接，发现 Hero Section 文案太技术化
    ↓
用户：在任务评论中反馈："把'基于机器学习算法'改成'懂你的 AI 教练'"
    ↓
Agent：自动更新代码并重新部署
    ↓
迭代计数：1/5
```

**4. 上线与追踪**

```
Agent：部署到生产环境，返回正式域名
    ↓
用户：在 Canvas 上将"Landing Page"节点标记为"已完成"
    ↓
系统：自动创建新任务"追踪 Waitlist 转化率"
```

#### 预期产出

- 一个可访问的 Landing Page（fitai-waitlist.vercel.app）
- 完整的源代码（保存在 Workspace 容器中）
- 部署文档（自动生成）
- 邮件收集系统（已配置）

### 场景 3：团队协作（Iteration）

#### 用户故事

> "作为一个 2 人创业团队，我们希望在快速迭代时保持战略一致，并且能清楚地追踪每个任务的进展。"

#### 详细流程

**1. 周会战略对齐（30 分钟）**

```
创始人 A：在 Canvas 上更新本周 OKR
    - O1：提升 Landing Page 转化率到 15%
    - KR1：优化 Hero Section 文案（负责人：A）
    - KR2：添加社交证明（负责人：B）
    - KR3：A/B 测试 CTA 按钮颜色（负责人：Agent）
    ↓
系统：自动将每个 KR 转化为任务中心的 Task
    ↓
AI：基于历史数据，预测任务工作量和完成时间
```

**2. 日常任务管理**

```
创始人 B：在任务中心查看"本周我的任务"
    ↓
创始人 B：发现"添加社交证明"任务没有明确验收标准
    ↓
创始人 B：在任务评论中 @AI："帮我写一下这个任务的验收标准"
    ↓
AI：自动生成
    - [ ] 至少展示 3 条用户评价
    - [ ] 评价包含头像、姓名、具体反馈
    - [ ] 支持移动端自适应
    - [ ] 添加"查看更多评价"链接（指向 Product Hunt）
    ↓
创始人 B：确认后开始执行
```

**3. Agent 自动执行任务**

```
系统：检测到"A/B 测试 CTA 按钮颜色"分配给 Agent
    ↓
Agent：自动执行
    - 分析当前页面代码
    - 创建 2 个变体（橙色按钮 vs 蓝色按钮）
    - 集成 Google Optimize 或 Vercel A/B Testing
    - 生成测试配置代码
    ↓
Agent：在任务评论中更新进度："已完成代码修改，等待你确认后部署"
    ↓
创始人 A：审查代码，批准部署
    ↓
Agent：自动部署，任务状态更新为"已完成"
```

**4. 周末复盘**

```
团队：查看 Canvas，发现 O1 进度为 80%
    ↓
AI：自动生成本周总结
    - ✅ 完成任务：5/6
    - ⏳ 阻塞任务：1（等待 A/B 测试数据）
    - 💡 关键洞察：社交证明功能上线后，转化率提升 8%
    - 📋 下周建议：继续优化文案，雇佣"SEO Optimization Agent"
    ↓
团队：基于总结调整下周计划
```

#### 预期产出

- Canvas 上 OKR 进度清晰可见
- 任务中心中所有任务都有负责人和状态
- 每周自动生成复盘报告
- 团队协作效率提升 40%

## 用户需求优先级

### P0（必须满足）

1. **Solo Entrepreneur**：
   - 快速整理想法的能力
   - AI 辅助调研
   - 生成基础 MVP 原型
   - 简单直观的任务管理

2. **小型团队**：
   - 实时协作（Canvas 同步）
   - 任务分配和追踪
   - 自动化文档生成
   - 进度可视化

### P1（重要但可延后）

1. **Solo Entrepreneur**：
   - 更多 Agent 服务选择
   - 模板库扩展
   - 导出功能（PDF/PNG）
   - 移动端支持

2. **小型团队**：
   - 团队角色和权限
   - 集成更多第三方工具（GitHub、Slack）
   - 高级分析和报表
   - 自定义工作流

### P2（Nice to have）

1. **Solo Entrepreneur**：
   - AI 自动推荐下一步行动
   - 社区功能（分享想法、找合作者）
   - 融资准备工具包

2. **小型团队**：
   - 白标部署
   - 企业级 SSO
   - 审计日志
   - API 开放平台
