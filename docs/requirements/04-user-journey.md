## 新用户首次体验（Onboarding）

### 目标

- 让用户在 **10 分钟**内理解 CrossMind 的核心价值
- 引导用户完成第一个"想法 → 调研"的完整流程
- 降低学习曲线，避免"功能太多不知道从哪开始"

### 详细流程

#### Step 1: 注册登录（1 分钟）

```
用户：访问 crossmind.app
    ↓
系统：展示 Hero Section
    - 标题："From Idea to Product, in Days not Months"
    - 副标题："AI-powered platform for solo entrepreneurs"
    - CTA："Start for Free" / "Sign in with Google"
    ↓
用户：点击"Sign in with Google"
    ↓
系统：OAuth 认证
    ↓
系统：创建默认项目"我的第一个项目"
    ↓
系统：跳转到欢迎页面
```

#### Step 2: 欢迎引导（2 分钟）

```
系统：展示欢迎页面（全屏模态窗口）
    ├─ 标题："欢迎来到 CrossMind 🎉"
    ├─ 3 步引导动画（自动播放，每步 3 秒）
    │   ├─ 动画 1："Canvas：像白板一样整理你的想法"
    │   │   └─ 展示节点创建、连线、AI 对话的动画
    │   ├─ 动画 2："AI：随时随地获得智能建议"
    │   │   └─ 展示 AI 自动填充、拆解任务的动画
    │   └─ 动画 3："Agent：让 AI 帮你完成任务"
    │       └─ 展示雇佣 Agent、自动执行的动画
    └─ CTA："开始我的第一个项目"
    ↓
用户：点击"开始我的第一个项目"
    ↓
系统：关闭模态窗口，进入 Canvas 页面
```

#### Step 3: 创建第一个想法（3 分钟）

```
    └─ 内容："AI 驱动的健身教练 App

    ↓
系统：右侧自动打开对话框（带淡入动画）
    ↓

    ↓
AI：分析回答，自动在 Canvas 上创建 2 个子节点（带动画）
    ├─ 节点 1："目标用户：健身新手"
    └─ 节点 2："核心痛点：缺乏个性化指导"
    ↓

    "太棒了！你已经完成了想法的初步结构化 ✨
```

```
        "💡 小技巧：
         - 双击空白区域创建新节点
         - 选中节点后按 Delete 删除
         - 点击节点右上角 ✨ 激活 AI 辅助
         - 按住 Space 键拖动 Canvas"
```

### 关键设计点

1. **示例数据**
   - 提供一个预填充的示例项目，用户可以直接体验
   - 避免"空白页焦虑"
   - 用户可以随时替换为自己的想法

2. **分步引导**
   - 不一次性展示所有功能
   - 按照自然流程逐步解锁（想法 → AI → Agent）
   - 每一步都有明确的目标和反馈

3. **即时反馈**
   - 每完成一步都有视觉反馈（动画、提示气泡、进度条）
   - Agent 工作时提供实时日志
   - 完成后有成就感展示

4. **免费首体验**
   - 第一次雇佣 Agent 免费
   - 降低尝试门槛
   - 让用户快速看到价值

5. **可跳过设计**
   - 每个引导步骤都可以跳过
   - 提供"不再显示"选项
   - 在帮助中心可以重新观看

## 关键设计要点

### 1. 实时协作

- **WebSocket 同步**：多人同时编辑 Canvas，实时看到对方的操作
- **在线状态**：显示哪些成员在线，正在查看哪个页面
- **协作光标**：显示其他成员的鼠标位置和选中节点
- **冲突处理**：同时编辑同一节点时，使用 CRDT 算法合并变更

### 2. 通知系统

- **分级通知**：
  - 高优先级：任务分配、@提醒、阻塞问题 → 推送通知 + 邮件
  - 中优先级：任务状态变更、评论回复 → 推送通知
  - 低优先级：系统更新、功能推荐 → 仅通知中心

- **通知聚合**：
  - 同一任务的多次状态变更 → 聚合为一条
  - 同一用户的多次 @提醒 → 聚合为一条

- **通知设置**：
  - 用户可以自定义通知偏好（哪些事件发送邮件）
  - 支持"勿扰模式"（晚上 10 点 - 早上 8 点）

### 3. 移动端优化

- **响应式设计**：所有页面支持桌面/平板/手机
- **手势操作**：
  - Canvas：双指缩放、拖动
  - 任务看板：左滑删除、右滑标记完成
- **离线支持**：
  - 缓存最近查看的项目和任务
  - 离线时可以查看、评论（同步后上传）
- **推送通知**：支持 PWA 推送通知

### 4. 性能优化

- **Canvas 渲染**：
  - 虚拟化渲染（只渲染可视区域内的节点）
  - 节点数量 > 100 时，自动降低连线渲染精度
  - 支持"简化模式"（只显示节点标题，隐藏内容）

- **增量加载**：
  - 任务列表分页加载（每页 50 条）
  - Canvas 节点懒加载（按需加载节点内容）
  - 评论和 Activity Feed 分页加载

- **缓存策略**：
  - Canvas 数据缓存 5 分钟
  - 任务列表缓存 1 分钟
  - 静态资源（图片、CSS、JS）CDN 缓存
