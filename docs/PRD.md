## 1. 产品概述

### 1.1 产品定义

**CrossMind 是什么？**

CrossMind 是一个 AI 驱动的全流程产品孵化平台，专为 solo entrepreneurs 和小型创业团队设计。它通过结构化的思维框架和自动化的 AI 能力，帮助用户将模糊想法转化为可落地的产品。

**核心理念：**

```
想法（Ideation）→ 调研（Research）→ 原型（Prototype）→ 迭代（Iteration）
```

### 1.2 核心价值主张

**对用户的价值：**

1. **降低门槛**：无需技术背景，AI 辅助完成从想法到原型的全流程
2. **提升效率**：自动化研究、设计、开发等重复性工作，节省 70% 时间
3. **结构化思维**：预设方法论框架，避免"不知道从哪开始"的困境
4. **持续迭代**：项目全周期追踪，确保从想法到产品的每一步都有据可查

**对比传统方式：**

| 传统方式                       | CrossMind 方式               |
| ------------------------------ | ---------------------------- |
| 想法散落在笔记本、便签、脑海中 | 统一在 Canvas 中结构化呈现   |
| 手动搜索竞品、用户反馈         | AI Agent 自动调研并生成报告  |
| 需要雇佣设计师、开发者         | Agent 雇佣中心按需购买服务   |
| 任务管理和执行分离             | 任务中心与 AI 执行无缝集成   |
| 文档散落各处，难以追溯         | 知识库自动归档，RAG 随时调用 |

### 1.3 差异化优势

**三个核心差异点：**

1. **Canvas + AI 的深度融合**

   - 不是简单的聊天机器人，而是"可视化思维空间 + AI 助手"
   - AI 直接操作 Canvas 节点，而非只是生成文本

2. **Agent 即服务（Agent-as-a-Service）**

   - 标准化的 AI 服务市场，按结果付费
   - 降低使用门槛，无需学习复杂的 prompt engineering

3. **Workspace 容器隔离架构**
   - 真正的多租户隔离，安全性高
   - 支持深度任务（文件操作、代码执行、外部集成）

## 2. 用户画像与场景

> 详细内容见：[用户画像与场景](./requirements/03-user-personas.md)

### 2.1 主要用户群（Primary Persona）

**角色：Solo Entrepreneur - Sarah**

**基本信息：**

- 年龄：28-35 岁
- 职业背景：产品经理、设计师、营销人员
- 技术水平：懂基本概念，但不会编程
- 创业阶段：0-1 阶段，想法验证期

**痛点：**

1. 有很多想法，但不知道如何系统化整理
2. 需要做市场调研，但没时间和精力手动搜集
3. 想做 MVP，但雇佣开发团队成本太高
4. 项目管理工具太复杂，一个人用 Jira 感觉"杀鸡用牛刀"

**目标：**

- 6 个月内验证 3 个想法
- 找到 PMF 后再考虑融资或招人
- 希望把更多时间花在"思考"而非"执行"上

### 2.2 次要用户群（Secondary Persona）

**角色：小型创业团队 - TechBros**

**基本信息：**

- 团队规模：2-5 人
- 角色分工：技术创始人 + 产品/运营
- 技术水平：至少有一人懂开发
- 创业阶段：已有 MVP，进入快速迭代期

**痛点：**

1. 团队小，每个人身兼多职，时间不够用
2. 需要频繁调整产品方向，但文档更新跟不上
3. 想尝试很多增长策略，但执行力有限
4. 协作工具多了之后，信息散落各处

## 3. 核心功能需求

> 详细内容见：[产品功能模块](./requirements/02-features.md)

### 3.1 Canvas - 战略思维空间

#### 基础能力

| 功能模块     | 详细描述                                                  | 优先级 |
| ------------ | --------------------------------------------------------- | ------ |
| **节点管理** | 创建、编辑、删除、移动节点                                | P0     |
| **节点类型** | Idea / Research / Prototype / Decision / Task / Link      | P0     |
| **层级结构** | 支持父子关系，最多 5 层                                   | P0     |
| **连线**     | 节点之间可以建立关系（依赖、支持、冲突）                  | P1     |
| **分组**     | 用虚线框将相关节点分组                                    | P1     |
| **标签**     | 每个节点支持多标签（如 #urgent #blocked #needs-research） | P0     |

#### AI 增强

| 功能模块     | 详细描述                                           | 优先级 |
| ------------ | -------------------------------------------------- | ------ |
| **智能建议** | AI 扫描 Canvas，提示缺失模块                       | P0     |
| **自动拆解** | 将大节点自动拆解为子节点                           | P1     |
| **关系推理** | AI 推荐节点之间的关系连线                          | P2     |
| **模板应用** | 一键应用预设框架（商业模式画布、精益画布、OKR 等） | P0     |

#### 预设模板库

- **商业模式画布**（Business Model Canvas）
- **OKR 框架**
- **产品战略分解**（Product Strategy Map）
- **精益画布**（Lean Canvas）

---

### 3.2 任务中心 - 项目操作系统

#### 任务管理

| 功能模块     | 详细描述                                                   | 优先级 |
| ------------ | ---------------------------------------------------------- | ------ |
| **创建任务** | 手动创建 / AI 从对话中抽取 / Canvas 节点转化               | P0     |
| **任务字段** | 标题、描述、状态、优先级、负责人、标签、截止日期、验收标准 | P0     |
| **状态流转** | Todo → In Progress → Review → Done / Blocked / Cancelled   | P0     |
| **子任务**   | 支持任务拆解为子任务（最多 3 层）                          | P1     |
| **依赖关系** | 任务 A 依赖任务 B（被依赖任务完成前，不能开始）            | P1     |
| **批量操作** | 批量修改状态、负责人、标签                                 | P1     |

#### 标签系统设计

**命名空间（Namespace）体系：**

CrossMind 采用 `namespace:value` 的标签结构，支持多维度分类。

| Namespace     | 示例值                                                               | 用途             |
| ------------- | -------------------------------------------------------------------- | ---------------- |
| `stage`       | `stage/research` `stage/design` `stage/dev` `stage/launch`           | 项目阶段         |
| `priority`    | `priority/critical` `priority/high` `priority/medium` `priority/low` | 优先级           |
| `risk`        | `risk/high` `risk/blocked`                                           | 风险标记         |
| `skill`       | `skill/design` `skill/frontend` `skill/backend` `skill/marketing`    | 所需技能         |
| `type`        | `type/bug` `type/feature` `type/doc` `type/research`                 | 任务类型         |
| `integration` | `integration/github` `integration/stripe` `integration/vercel`       | 涉及的第三方集成 |

### 3.3 Agent 雇佣中心 - AI 服务市场

#### 服务分类

| 阶段         | 服务类型 | 具体服务示例                                           |
| ------------ | -------- | ------------------------------------------------------ |
| **想法验证** | 市场调研 | Reddit 调研、Product Hunt 竞品分析、Google Trends 分析 |
|              | 用户访谈 | 自动生成访谈问题、整理访谈记录、提取关键洞察           |
| **设计阶段** | 头脑风暴 | AI 辅助 brainstorming、商业模式画布生成                |
|              | 需求设计 | 生成 PRD、用户故事、用例图                             |
|              | 原型设计 | Landing Page 生成、Figma 原型建议                      |
| **开发阶段** | 代码生成 | Next.js App 脚手架、API 接口生成、数据库 Schema 设计   |
|              | 技术选型 | 技术栈对比报告、架构设计建议                           |
|              | 测试     | 单元测试生成、E2E 测试脚本                             |
| **运营阶段** | 内容创作 | 博客文章、社交媒体帖子、SEO 优化                       |
|              | 营销活动 | Campaign 策划、A/B 测试方案、增长实验设计              |
|              | 数据分析 | GA4 数据解读、用户行为分析、转化漏斗优化建议           |

#### 服务流程

```
用户下单
    ↓
填写输入表单（关键词、Subreddit、时间范围）
    ↓
授权相关 API（如果需要）
    ↓
确认订单（支付费用）
    ↓
Agent 开始工作（后台执行）
    ↓
任务中心生成任务追踪
    ↓
Agent 完成第一版产出
    ↓
用户收到通知，查看结果
    ↓
[可选] 用户提供反馈
    ↓
Agent 迭代更新（迭代计数）
    ↓
用户满意后标记"完成"
    ↓
任务状态更新为 Done
    ↓
用户评价（可选，匿名）
```

#### 反馈与迭代机制

**迭代次数限制设计：**

| 服务类型                            | 基础迭代次数 | 超出后价格 | 理由                         |
| ----------------------------------- | ------------ | ---------- | ---------------------------- |
| 简单调研（Reddit、Product Hunt）    | 3 次         | $9/次      | 主要是数据获取，迭代空间小   |
| 文档生成（PRD、PPT、设计文档）      | 5 次         | $15/次     | 需要多次调整格式和内容       |
| 代码生成（Landing Page、API）       | 5 次         | $19/次     | 需要测试和调整细节           |
| 复杂任务（架构设计、Campaign 策划） | 7 次         | $29/次     | 涉及多方面决策，需要充分讨论 |

## 4. 用户旅程设计

> 详细内容见：[用户旅程设计](./requirements/04-user-journey.md)

### 4.1 新用户首次体验（Onboarding）

**目标：**

- 让用户在 10 分钟内理解 CrossMind 的核心价值
- 引导用户完成第一个"想法 → 调研"的完整流程
- 降低学习曲线，避免"功能太多不知道从哪开始"

**关键设计点：**

1. **示例数据**：提供一个预填充的示例项目，用户可以直接体验
2. **分步引导**：不一次性展示所有功能，而是按照自然流程逐步解锁
3. **即时反馈**：每完成一步都有视觉反馈（动画、提示气泡）
4. **免费首体验**：第一次雇佣 Agent 免费，降低尝试门槛

### 4.2 核心使用场景

#### 场景 1：想法孵化（Ideation → Research）

**用户故事：**

> "作为一个有想法的 solo entrepreneur，我希望能够快速验证我的想法是否值得投入时间，所以我需要一个工具帮我整理思路并做初步市场调研。"

**预期产出：**

- Canvas 上有 15-20 个结构化节点
- 一份 5 页的调研报告（PDF）
- 任务中心中自动生成 3-5 个后续任务

#### 场景 2：原型搭建（Prototype）

**用户故事：**

> "作为一个非技术创始人,我希望能快速搭建一个 MVP 来收集用户反馈，所以我需要 AI 帮我生成 Landing Page 和基础功能原型。"

**预期产出：**

- 一个可访问的 Landing Page
- 完整的源代码（保存在 Workspace 容器中）
- 部署文档（自动生成）

#### 场景 3：团队协作（Iteration）

**用户故事：**

> "作为一个 2 人创业团队，我们希望在快速迭代时保持战略一致，并且能清楚地追踪每个任务的进展。"

**预期产出：**

- Canvas 上 OKR 进度清晰可见
- 任务中心中所有任务都有负责人和状态
- 每周自动生成复盘报告

## 5. 功能优先级与迭代计划

> 详细内容见：[功能优先级与迭代计划](./requirements/05-roadmap.md)

### 5.1 MVP 范围（P0 - 必须有）

**目标：**
用户能够完成"想法 → 调研 → 原型"的基本流程，体验到 AI 的核心价值。

**功能清单：**

| 模块               | 功能                                                | 预估工作量 |
| ------------------ | --------------------------------------------------- | ---------- |
| **用户系统**       | OAuth 登录（Google / GitHub）                       | 3 天       |
|                    | 项目创建与管理                                      | 2 天       |
|                    | 成员邀请（最多 5 人）                               | 3 天       |
| **Canvas**         | 基础节点 CRUD                                       | 5 天       |
|                    | 节点类型（Idea / Research / Prototype / Task）      | 2 天       |
|                    | 层级结构（父子关系）                                | 3 天       |
|                    | Markdown 编辑器                                     | 2 天       |
|                    | AI 对话面板                                         | 3 天       |
| **任务中心**       | 任务 CRUD                                           | 4 天       |
|                    | 看板视图                                            | 3 天       |
|                    | 状态流转                                            | 2 天       |
|                    | 评论功能                                            | 3 天       |
|                    | @ 提醒                                              | 2 天       |
| **Agent 雇佣中心** | 服务列表页                                          | 2 天       |
|                    | 服务详情页                                          | 3 天       |
|                    | 下单流程                                            | 4 天       |
|                    | 3 个初始服务（Reddit 调研、Landing Page、PRD 生成） | 10 天      |
| **AI 能力**        | 会话层（streamText + useChat）                      | 4 天       |
|                    | 自建 Provider                                       | 5 天       |
|                    | 基础 MCP 工具（create_task, update_canvas_node）    | 4 天       |
| **Workspace 容器** | 容器基础架构                                        | 7 天       |
|                    | Agent SDK 集成                                      | 5 天       |
|                    | 文件存储（Blob）                                    | 3 天       |

**总计：约 80 天（3-4 个月，1-2 人团队）**

## 6. 非功能需求

### 6.1 性能指标

- **响应速度**：AI 生成过程需有明确的 Loading 状态和进度提示
- **页面加载**：首屏渲染时间 < 2 秒
- **Canvas 流畅度**：支持 100+ 节点无卡顿

### 6.2 安全性

- **数据加密**：用户创意数据需加密存储
- **隐私保护**：不用于非授权的模型训练
- **权限控制**：项目级别的访问控制
- **容器隔离**：每个项目独立的 Workspace 容器

### 6.3 易用性

- **界面风格**：简洁现代，操作流程线性直观
- **多端支持**：响应式设计，支持桌面和平板
- **无障碍性**：遵循 WCAG 2.1 标准

### 6.4 可扩展性

- **水平扩展**：支持多实例部署
- **容器编排**：支持 Kubernetes 部署
- **数据库优化**：支持连接池和查询优化
