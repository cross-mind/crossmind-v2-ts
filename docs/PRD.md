## 1. 产品概述

CrossMind 是一个全流程产品孵化平台，旨在帮助用户将模糊的初步想法转化为可执行的落地项目。通过结构化的引导、AI 辅助生成和自动化管理，打通从概念到 MVP 再到研发执行的全链路。

## 2. 核心用户流程

### 2.1 第一阶段：想法澄清与文档生成 (Idea to Doc)
**目标**：将一句话或简单的描述转化为结构化的项目概要文档。

1.  **用户输入**：
    *   用户在输入框中描述产品想法（支持文本、语音转文字）。
    *   可选上传辅助材料（图片、草图）。
2.  **AI 分析与生成**：
    *   系统基于输入，生成包含以下模块的概要文档：
        *   **背景与痛点 (Background & Pain Points)**：市场环境、用户核心问题。
        *   **解决方案 (Solution)**：产品核心功能、解决方式。
        *   **MVP 要素 (MVP Scope)**：最小可行性产品的核心功能列表。
        *   **商业模式 (Business Model)**：潜在的盈利模式和目标用户群。
3.  **评分与反馈**：
    *   AI 对生成的文档进行质量评分（颗粒度、逻辑性、完整性）。
    *   如果评分低于阈值，提示用户补充信息或手动优化。
4.  **用户确认**：
    *   用户查看、编辑并确认文档内容。
    *   确认后进入下一阶段。

### 2.2 第二阶段：任务拆解与管理 (Doc to Tasks)
**目标**：将确认的文档转化为具体的执行任务。

1.  **MVP 启动**：
    *   用户点击“启动 MVP”按钮。
2.  **任务自动拆解**：
    *   AI 根据文档内容，生成分层级的任务列表。
    *   任务分类：
        *   **研发 (Development)**：前端、后端、基础设施。
        *   **设计 (Design)**：UI/UX、视觉素材。
        *   **市场/运营 (Marketing/Ops)**：竞品分析、用户调研、Landing Page。
3.  **任务看板管理**：
    *   提供看板视图 (Kanban Board)：待办 (Todo)、进行中 (In Progress)、已完成 (Done)。
    *   任务详情：支持添加描述、截止日期、优先级、负责人。

### 2.3 第三阶段：研发管理与 Agent 集成 (Tasks to Code)
**目标**：辅助研发执行，利用 AI Agent 提效。

1.  **研发看板**：
    *   专门的视图聚焦于研发类任务。
2.  **GitHub 集成**：
    *   **自动建仓**：关联 GitHub 账号，一键创建项目仓库。
    *   **配置同步**：将项目描述、环境变量等同步至仓库。
3.  **Agent 协作**：
    *   **配置 Agent**：支持为项目配置 AI 编程助手 (如 Cursor, Cloud Code, Codex 等的模拟配置或集成)。
    *   **工作流 (Workflow)**：在仓库中生成 GitHub Actions，用于 CI/CD 或代码检查。
    *   **任务委派**：(未来规划) 将特定代码任务指派给 AI Agent 自动提交 PR。

## 4. 非功能需求
*   **响应速度**：AI 生成过程需有明确的 Loading 状态和进度提示。
*   **数据安全**：用户创意数据需加密存储，不用于非授权的模型训练。
*   **易用性**：界面风格简洁现代，操作流程线性直观。
